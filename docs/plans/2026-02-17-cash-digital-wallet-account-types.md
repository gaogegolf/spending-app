# Cash & Digital Wallet Account Types

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add `CASH` and `DIGITAL_WALLET` account types so users can track physical cash and P2P/digital wallet transactions (Venmo, PayPal, Zelle, etc.).

**Architecture:** Add two new values to the backend `AccountType` enum with an Alembic migration, then update the frontend type union, display arrays, labels, and icons across all pages that reference account types.

**Tech Stack:** Python/SQLAlchemy (backend enum + migration), TypeScript/Next.js (frontend types + UI)

**Files:**
- Modify: `backend/app/models/account.py:12-26`

**Step 1: Add enum values to AccountType**

In `backend/app/models/account.py`, add two values to the `AccountType` enum after `OTHER`:

```python
class AccountType(str, enum.Enum):
    """Types of accounts."""

    # Traditional banking
    CREDIT_CARD = "CREDIT_CARD"
    CHECKING = "CHECKING"
    SAVINGS = "SAVINGS"
    INVESTMENT = "INVESTMENT"
    OTHER = "OTHER"
    CASH = "CASH"
    DIGITAL_WALLET = "DIGITAL_WALLET"
    # Brokerage accounts
    BROKERAGE = "BROKERAGE"  # Taxable brokerage
    IRA_ROTH = "IRA_ROTH"  # Roth IRA
    IRA_TRADITIONAL = "IRA_TRADITIONAL"  # Traditional IRA
    RETIREMENT_401K = "RETIREMENT_401K"  # 401(k) plans
    STOCK_PLAN = "STOCK_PLAN"  # RSU/ESPP plans
```

**Step 2: Generate Alembic migration**

Run:
```bash
cd backend && source venv/bin/activate && alembic revision --autogenerate -m "add cash and digital wallet account types"
```

The autogenerated migration won't handle enum changes correctly for SQLite. Manually edit the migration to use raw SQL:

```python
def upgrade() -> None:
    # SQLite doesn't enforce enum constraints, so no DDL needed.
    # The new values are valid at the application layer via the Python enum.
    pass

def downgrade() -> None:
    pass
```

**Step 3: Run migration**

Run: `cd backend && source venv/bin/activate && alembic upgrade head`

**Step 4: Commit**

```bash

git commit -m "feat: add CASH and DIGITAL_WALLET account types to backend enum"
```
